## 4. Trous bloquant le signoff

1. **Outil de vérité indisponible ici** : `s5cmd`/`rclone`/`aws` absents → la
   liste live S3 (`scw:/graph/`) ne peut pas être régénérée dans cet
   environnement. C'est le blocage n°1 : sans elle, aucune preuve « fraîche ».
2. **Dernière mesure vérifiable = 145/1104 v2.3** (13 %). Très loin d'une
   complétude signable.
3. **97 villes éligibles n'ont aucun graphe S3** et **ne figuraient même pas**
   dans l'union ciblée v23 (1009 < 1104) — elles n'ont jamais été traitées.
   Échantillon : `amqui, audet, baie-johan-beetz, beaulac-garthby, beaumont,
   belcourt, belleterre, bethanie, bonne-esperance, bonsecours …` (97 au total,
   liste complète reproductible, cf. §5).
4. **862 villes encore en v2.2** au dernier audit (migration 2.3 non faite).
5. **Citations non auditées par ville** : la complétude « citations exhaustives »
   de 2.3 n'a pas de compteur par ville dans les artefacts locaux ; elle n'est
   garantie que par le gate au moment du publish, sur les seules villes publiées.

**Conclusion : signoff impossible.** Il n'existe aucune trace prouvant 2.3 sur
les ~1104 cibles ; la preuve disponible démontre l'inverse (large incomplétude).

## 5. Commande(s) reproductibles pour régénérer la mesure

Prérequis manquant : installer `s5cmd` (ou rclone/aws). Ensuite, mesure
autoritaire de complétude 2.3 contre S3 :

```bash
cd /home/antoinefa/src/radar-immobilier
set -a; . ./.env; set +a
export AWS_ACCESS_KEY_ID="$SCRAPE_S3_ACCESS_KEY" \
       AWS_SECRET_ACCESS_KEY="$SCRAPE_S3_SECRET_KEY"
EP="$SCRAPE_S3_ENDPOINT"; B="$SCRAPE_S3_BUCKET"

# 1) toutes les villes ayant un graphe publié sur S3
s5cmd --endpoint-url "$EP" ls "s3://$B/graph/" \
  | awk '{print $NF}' | sed 's#/##' | sort -u > /tmp/graph-cities.txt

# 2) ontology_version par ville (vérité par ville)
: > /tmp/graph-versions.tsv
while read -r c; do
  v=$(s5cmd --endpoint-url "$EP" cat "s3://$B/graph/$c/latest.json" 2>/dev/null \
       | jq -r '.ontology_version // "?"')
  printf '%s\t%s\n' "$c" "$v" >> /tmp/graph-versions.tsv
done < /tmp/graph-cities.txt

# 3) cible éligible (1104) depuis la source canonique
node -e "const m=require('./packages/radar-sources/src/geo/municipalities.qc.json');\
console.log(m.filter(x=>!x.excluded).map(x=>x.slug).sort().join('\n'))" > /tmp/eligible.txt

# 4) complétude + liste exacte des trous
LC_ALL=C; awk -F'\t' '$2=="2.3"{print $1}' /tmp/graph-versions.tsv | sort -u > /tmp/v23.txt
echo "v2.3=$(wc -l < /tmp/v23.txt) / cible=$(wc -l < /tmp/eligible.txt)"
comm -23 /tmp/eligible.txt /tmp/v23.txt > /tmp/missing-v23.txt   # villes à finir
echo "manquantes=$(wc -l < /tmp/missing-v23.txt)"
```

Pour relancer le traitement des villes manquantes (publication 2.3) :
```bash
cd /home/antoinefa/src/radar-immobilier
bash tools/graphify-v23/preflight.sh        # vérifie s5cmd + accès S3
bash tools/graphify-v23/runner.sh           # run complet (sans --dry-run)
```
