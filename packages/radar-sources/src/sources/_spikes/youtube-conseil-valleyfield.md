## 4. Transcription — résultat et bloqueurs documentés

### 4a. Méthodes tentées

| Méthode | Résultat |
|---|---|
| `WebFetch` sur `youtube.com/watch?v=YDhtxnEwSnM` | Page JS-rendue — seul le footer HTML retourné, aucun contenu vidéo |
| `youtube.com/api/timedtext?lang=fr&v=YDhtxnEwSnM` | Corps vide — endpoint exige des cookies/session YouTube |
| `video.google.com/timedtext?lang=fr&v=YDhtxnEwSnM` | Corps vide — même contrainte |
| `youtubetranscript.com/?v=YDhtxnEwSnM` | HTTP 403 Forbidden |
| `downsub.com` | Page de service seulement, pas de transcript |

**Conclusion :** Transcript NON OBTENU via les outils disponibles (WebFetch/WebSearch).

### 4b. Extrait de transcription

Information non disponible.

**Bloqueur exact :** L'API timedtext de YouTube requiert un cookie de session authentifié (`CONSENT`, `VISITOR_INFO1_LIVE`, etc.) que WebFetch ne peut pas fournir. Les pages vidéo YouTube sont rendues côté client par JavaScript — le HTML brut retourné ne contient pas les données de transcript. Les services tiers (youtubetranscript.com, downsub.com) ont bloqué les requêtes sans navigateur réel (HTTP 403).

### 4c. Méthode qui fonctionnerait (Tier-B-partial)

```
# Option 1 — yt-dlp (en container)
docker run --rm yt-dlp/yt-dlp \
  --write-auto-sub --sub-lang fr --skip-download \
  --output "/out/%(id)s.%(ext)s" \
  "https://www.youtube.com/watch?v=YDhtxnEwSnM"
# Produit : YDhtxnEwSnM.fr.vtt (sous-titres auto-générés)

# Option 2 — Si pas de captions auto
docker run --rm yt-dlp/yt-dlp \
  --extract-audio --audio-format mp3 \
  "https://www.youtube.com/watch?v=YDhtxnEwSnM" | \
  whisper --language fr --model medium -

# Option 3 — Playwright (navigateur réel)
# Se connecter à Obscura (obscura://obscura:9222), naviguer vers la page
# vidéo, cliquer "..." > Ouvrir le transcript, extraire le texte du DOM
```

Les coûts de transcription Whisper (cloud) pour une séance de ~3h sont d'environ 1,80 USD (OpenAI API) ou ~0 avec un modèle local medium sur GPU.
